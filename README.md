Здравствуй, товарищ! Это сайт канала [Дилетантская Озвучка](https://t.me/dilettante_voiceover), тут мы систематизируем всё, что мы поназаписывали за долгие годы.
Ниже инструкция как это всё обновлять, на случай, если всех нас собъёт автобус. 
Так же, если тебе тоже нужен сайтик для своих либертарных целей, можешь пользоваться нашим как примером.
Сделать такой и захостить его на GitHub Pages очень легко.

# Как этим пользоваться

Контент хранится в трёх YAML-файлах в папке `_data/`. Их и нужно редактировать.

## Добавить новую запись

Открой `_data/recordings.yml` и добавь блок в нужное место:

```yaml
- id: уникальный-идентификатор
  title: "Название"
  author: "Автор"          # или ~ если автора нет
  url: "https://t.me/dilettante_voiceover/XXX"
  type: book               # см. типы ниже
  status: complete         # только для книг: complete или in_progress
  topics: [likbez, prison] # теги из _data/topics.yml
```

**Типы записей** (`type`):
- `book` — книга целиком или в работе (различаются полем `status: complete` / `status: in_progress`)
- `book_excerpt` — отрывок из книги
- `article` — статья или эссе (нужно также указать `source`, см. ниже)
- `art` — художественный текст
- `history` — исторический источник

**Для статей** нужно добавить поле `source` — название источника, как оно прописано в `_data/sections.yml`:

```yaml
  source: "Crimethinc"
```

## Добавить новый тег

Открой `_data/topics.yml` и добавь строку:

```yaml
- id: my-topic
  label: "Название тега по-русски"
```

Затем используй `id` в поле `topics` в `_data/recordings.yml`.

## Добавить новый раздел или подраздел

Разделы и подразделы настраиваются в `_data/sections.yml`. Порядок записей в файле определяет порядок на странице.

Раздел статей с подразделами по источникам выглядит так:

```yaml
- id: essay
  title: "Отдельные статьи, эссе"
  type: article
  subsections:
    - id: crimethinc
      title: "Сrimethinc"
      source: "Crimethinc"
```

## Как это сделано

Сайт работает на [Jekyll](https://jekyllrb.com/) и хостится на GitHub Pages. Пуш в ветку `main` автоматически обновляет сайт.

Локальный запуск:

```bash
bundle install
bundle exec jekyll serve
```
